## Gabay sa Paggamit

### Gumawa ng Website App
1. I-click ang "Gumawa ng App" na button sa home screen
2. Ilagay ang pangalan ng app at website URL
3. (Opsyonal) Pumili ng custom icon o gumawa gamit ang AI
4. (Opsyonal) I-configure ang splash screen, BGM, activation code, anunsyo, ad blocking, atbp.
5. (Opsyonal) Pumili ng extension modules para palakasin ang functionality
6. I-click ang Save

### Gumawa ng Media App
1. I-click ang "Gumawa ng Media App" na button
2. Pumili ng image o video file
3. I-configure ang display options (loop, autoplay, audio toggle, atbp.)
4. (Opsyonal) Magdagdag ng background music
5. I-save at i-build ang APK

### Gumawa ng HTML App
1. I-click ang "Gumawa ng HTML App" na button
2. Pumili ng HTML project folder o single file
3. I-set ang entry file (default index.html)
4. Suporta sa React/Vue/Next.js build outputs
5. I-save at i-build ang APK

### Paggamit ng Extension Modules
1. Kapag gumagawa/nag-e-edit ng app, i-expand ang "Extension Modules" card
2. I-click ang "Pumili ng Modules" para mag-browse ng 10 built-in modules
3. Pumili ng mga gustong module (video download, dark mode, privacy protection, atbp.)
4. Awtomatikong mag-i-inject at mag-e-execute ang mga module kapag tumatakbo ang app

### AI-Assisted Development
1. Pumunta sa "Extension Modules" > "AI Module Development"
2. Ilarawan ang functionality na gusto mo sa natural language
3. Gagawa ang AI ng module code na may syntax check at security scan
4. I-preview ang epekto at i-save

### Pagpapatakbo ng App
- I-click ang app card para direktang mag-preview
- Long press o i-click ang menu para sa iba pang mga opsyon

### Pag-build ng APK Package
1. I-click ang app card menu > "Build APK"
2. I-configure ang encryption options (opsyonal)
3. I-configure ang isolated browser environment (opsyonal)
4. I-configure ang background running (opsyonal)
5. I-click ang "Simulan ang Build"
6. Awtomatikong lalabas ang installation dialog kapag tapos na

### Pag-configure ng Forced Run Mode
1. Kapag gumagawa/nag-e-edit ng app, i-expand ang "Forced Run" card
2. I-enable ang forced run at pumili ng mode (fixed time/countdown/duration)
3. I-configure ang time period at active days
4. I-configure ang black tech features (opsyonal)
5. I-build ang APK at awtomatikong tatakbo ang app sa itinakdang oras

### Paggamit ng App Modifier
1. I-click ang home menu > "App Modifier"
2. Maghanap o mag-filter ng target app sa listahan ng app
3. I-click ang app para pumasok sa modification interface
4. Pumili ng bagong icon, ilagay ang bagong pangalan
5. Pumili ng operasyon:
   - **Shortcut**: Gumawa ng desktop shortcut na may bagong icon
   - **Clone Install**: Gumawa ng bagong APK at i-install bilang independent app

## I-build mula sa Source

### Mga Kinakailangan
- Android Studio Hedgehog (2023.1.1) o mas bago
- JDK 17
- Gradle 8.2

### Mga Hakbang sa Pag-build
```bash
# I-clone ang repository
git clone https://github.com/shiahonb777/web-to-app.git

# Pumasok sa project directory
cd web-to-app

# I-build ang Debug version
./gradlew assembleDebug

# I-build ang Release version
./gradlew assembleRelease
```

> **Paalala:** Kung nagbi-build mula sa command line (labas ng Android Studio), kailangan mong gumawa ng `local.properties` file sa project root na may Android SDK path mo:
> ```
> sdk.dir=/path/to/your/android/sdk
> ```

### Signing Configuration
Para sa Release builds, i-configure ang signing sa `app/build.gradle.kts`:
```kotlin
signingConfigs {
    create("release") {
        storeFile = file("your-keystore.jks")
        storePassword = "your-store-password"
        keyAlias = "your-key-alias"
        keyPassword = "your-key-password"
    }
}
```

> **Tip:** Para sa debug builds, hindi kailangan ng signing configuration — patakbuhin lang ang `./gradlew assembleDebug`.

## Mga Paalala

1. Ang ilang website ay maaaring may anti-crawling mechanisms, maaaring limitado ang pag-load
2. Kailangan ng network permission para sa normal na paggamit
3. Ang mga na-export na proyekto ay kailangang i-compile gamit ang Android Studio sa PC
4. Ang mga activation code ay lokal lamang ang verification, i-extend para sa server-side verification kung kinakailangan
5. Ang mga extension module ay nag-e-execute sa WebView, ang ilang sites ay maaaring may CSP restrictions
